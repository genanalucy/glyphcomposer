# newthesis 目录结构说明

## 说明

本文件用于说明当前工作区的整理结果与主要目录用途。  
本次整理遵循一个原则：**代码目录保持不动，论文材料集中收纳到 `docs/thesis/`**，从而减少仓库根目录的混杂感。

下方目录树省略了 `.git/`、`.venv/`、`__pycache__/`、`.pytest_cache/` 等运行缓存目录，只保留项目实际工作所需的主要结构。

## 各目录用途

### 根目录

- `README.md`：英文项目说明。
- `README.zh-CN.md`：中文项目说明。
- `app.py`：应用入口。
- `pyproject.toml`：项目构建与包配置。
- `requirements.txt`：依赖列表。

### `assets/`

- 存放静态样例资源。
- 当前包括拆字表示例和 OOV 样本模板。

### `configs/`

- 存放各训练任务的配置文件。
- 当前覆盖基线、VAE、潜空间扩散、结构探针和 OOV LoRA 适配。

### `data/`

- 用于放置运行过程中生成或整理后的数据。
- `data/generated/` 当前作为生成数据集输出目录。

### `docs/`

- 统一存放说明性文档，避免根目录混放资料文件。
- `docs/目录结构说明.md`：当前这份目录说明。
- `docs/thesis/`：论文相关文档目录。

### `docs/thesis/`

- `硕士开题报告.md`：当前硕士开题报告正文。
- `写作去AI味提示.md`：用于约束论文改写风格的写作提示。

### `scripts/`

- 存放数据准备、训练、评测、推理和 demo 启动脚本。
- 这些脚本是项目运行层的主要入口。

### `src/glyphgen/`

- 存放项目核心源码。
- `data/`：数据读取、渲染、清洗与清单定义。
- `models/`：基线模型、VAE、扩散模型、LoRA 等实现。
- `training/`：损失函数、检查点等训练逻辑。
- `eval/`：评测指标与结构探针相关逻辑。
- `utils/`：通用工具函数。
- 其余顶层模块负责配置、运行时封装、推理接口、结构类型与公共类型定义。

### `tests/`

- 存放基础测试代码。
- 当前覆盖拆解、结构与前向流程等核心行为。

## 本次整理结果

本次目录整理只做了下面这些调整：

1. 保留原有代码结构不变。
2. 将论文相关文档从仓库根目录移动到 `docs/thesis/`。
3. 新增本文件，作为项目目录结构与用途说明。

## 后续建议

- 后续若继续补充论文、答辩或写作规范文档，建议统一放入 `docs/thesis/`。
- 若后续出现更多非论文类文档，可在 `docs/` 下再细分为如 `docs/project/`、`docs/experiments/` 等子目录。
