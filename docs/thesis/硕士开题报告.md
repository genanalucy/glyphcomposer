# 基于部件与结构条件约束的方块字字形生成方法研究

## 二、选题依据

### 2.1 研究背景

方块字具有明显的构形规律。无论是现代汉字、历史文献中的异体字，还是古壮字等尚未被 Unicode 完整收录的字符，其书写形态往往都不是无规则的图案，而是由若干部件在特定空间结构下组合而成。随着数字人文、古籍整理、少数民族文字信息化以及教育技术的发展，面向复杂字形的自动生成与表示问题变得越来越实际。对于现代常用汉字，现成字体库可以解决大部分显示问题；但一旦面对生僻字、方志中的异体字、文献整理中临时出现的未登录字符，传统字体制作就会很快暴露出成本高、周期长、依赖人工经验等局限。尤其是在需要批量处理、快速试排、辅助标注或构形验证的场景下，仅靠手工绘制和静态字库已经很难支撑持续扩展的需求。

从技术角度看，字形问题长期被拆分在两个相对独立的方向中：一类工作关注“识别”，研究如何从图像中判断一个字是什么；另一类工作关注“字体迁移”或“风格生成”，研究如何在给定字体风格下把已有字符迁移到新样式。前者往往强调部件、偏旁、笔画等细粒度表征，后者则更强调风格一致性和视觉质量。相比之下，“给定两个部件和一个结构标签，由模型直接生成完整字形”这一任务并没有被充分系统化。这个问题与开放域文生图不同，它的输入空间更小，监督更明确，生成目标也不是自然图像，而是具有强结构约束的单字灰度图像。因此，它既保留了生成建模的挑战，又具有更清晰的研究边界，适合作为硕士阶段的方法研究课题来展开。

本研究拟将问题定义为“部件与结构条件约束的方块字字形生成”。在这一设定中，模型输入不再是自然语言描述，而是两个可解释的字形部件与一个二元结构标签，如左右、上下或包围关系；模型输出是一张完整的单字图像。以“犭 + 句 + 左右结构”为例，目标不是程序化拼接出“狗”字，而是让模型学习部件之间的空间关系、比例分配、轮廓衔接和整体风格，从而生成结构正确、视觉连贯的字图。这样的任务定义能够把汉字构形知识、条件生成模型和组合泛化问题自然地联系起来，为后续面向古壮字、异体字和其他未登录字符的扩展奠定统一接口。

### 2.2 研究目的与意义

本课题的直接目标，是建立一套面向方块字的结构化条件生成方法：输入部件和结构条件，输出完整字形图像，并在组合泛化场景下保持较好的结构正确性和可读性。与常规的字体迁移任务相比，这里更关注“从部件到整字”的生成机理，而不是“从一种字体到另一种字体”的风格映射；与传统规则拼接相比，这里又要求最终图像由模型生成，而不是由人工设计的排版规则直接合成。这种研究设定兼顾了学术性和落地性，一方面可以讨论部件级表征、空间布局建模、潜空间生成等核心方法问题，另一方面又能够形成可运行的数据、训练、评测和演示闭环。

在理论层面，本研究的意义集中体现在三个相互关联的方向。它把汉字部件学、构形学中的“可分解性”引入生成任务，把部件级先验从识别领域拓展到图像生成领域，有助于探索一种更细粒度、更可解释的字形表示方式[1-10]。它也把组合泛化问题带入字形生成场景，关注训练时见过部件和结构、却没有见过某些具体整字组合时，模型是否仍能生成合理结果。这一问题与近年来零样本汉字识别、显式部件建模和结构编码的研究趋势具有内在一致性[1-10]。与此同时，本课题为“结构先验如何进入扩散模型或潜空间生成模型”提供了一个相对可控的研究对象，相比开放域图像生成，这类问题更容易观察模型何时学到了布局，何时只是记住了外观模板[11-17]。

在应用层面，本研究同样具有现实价值。对现代汉字而言，部件条件生成可以为字库补字、字形教学、字典编纂辅助和字体设计提供新工具；对未登录字符、古籍异体字以及古壮字等扩展对象而言，这一框架有望降低样本稀缺带来的建模门槛。尤其是在古文字或民间文字整理场景中，很多字符并没有稳定的编码入口，但研究者往往能给出其构形描述或局部部件图像。如果模型能够同时支持文本部件输入和图像部件输入，那么同一套方法就可以从现代汉字主任务平滑迁移到 OOV 场景，为后续的字符整理、检索和数字展示提供技术支持[18-26]。

### 2.3 国内外研究现状

与本课题最直接相关的研究基础，首先来自围绕“部件、偏旁和结构表示”展开的汉字识别工作。较早的研究已经意识到，汉字并不是一个不可分割的类别标签，而是具有显著内部构成的复合符号。Zhang 等提出 Radical Analysis Network，将偏旁信息用于零样本印刷体汉字识别，证明了部件级分析可以在未见类别上带来更好的迁移能力[1]。随后，Wang 等、Cao 等分别从少样本手写识别和层级分解嵌入角度继续推进，说明仅依赖整字类别监督并不足以充分利用方块字内部规律[2-3]。近几年，围绕显式部件编码、部件自信息、笔画与偏旁联合分解、特征感知编码等问题，又出现了一批更关注组合关系的工作[6-10]。与此同时，Yu 等将 IDS 对齐引入中文文本识别，Hong 等则用 formation tree 强化字符表示，这些研究说明构形描述已经开始从“辅助标签”走向“核心表示”[4-5]。不过，这一方向的主流目标仍然是识别或表示学习，而非根据部件反向生成整字图像。

与之并行发展的，是字体迁移与字形生成研究。早期工作多围绕少样本字体生成展开，典型思路是将内容和风格拆分，在少量参考字的条件下生成同风格的新字符。Azadi 等提出 Multi-Content GAN，较早系统讨论了 few-shot 字体迁移中的内容共享与风格转移问题[27]。随后，Localized Style Representations、Multiple Localized Experts、Fine-Grained Local Styles 等研究不断加强局部风格建模，使模型不再只依赖全局风格向量，而是尝试捕获笔画局部差异与风格细节[28-32]。在中文场景下，FontTransformer 及后续工作进一步把高分辨率字形合成与结构表征结合起来，推动了中文字形生成质量的提升[31,33]。最近两年，Diff-Font、FontDiffuser、Generate Like Experts 等研究又把扩散模型引入字体生成，使结果在纹理细节和多步去噪稳定性上取得明显进展[34-36]。然而，现有字体生成研究大多默认“目标字符的身份已知”，重点是从若干参考字学习目标字体风格；也就是说，模型解决的是“怎么把这个字写成另一种风格”，而不是“给定部件和结构，能否生成一个此前未显式提供整字模板的目标字形”。这与本课题的任务设定仍有实质差异。

生成模型本身的发展，则为本课题提供了方法论上的支撑。自 VAE 提出以来，潜变量建模为复杂图像的可控生成提供了统一框架[11]。基于非平衡热力学的扩散思想及其后续的 DDPM、Improved DDPM、DDIM、SDE 形式化与潜空间扩散模型，则显著提升了高质量图像生成的稳定性和可扩展性[12-17]。这些方法在自然图像、图像编辑和条件生成中已经形成较成熟的技术路线，但它们面向的多数对象仍然是自然图像或大尺度视觉概念。把这类模型直接用于字形生成，会遇到两个更细的问题：一是字符图像对局部结构错误极其敏感，哪怕只有局部部件位置偏移，也可能导致整字不可读；二是开放域提示词式条件并不适合方块字构形约束，模型需要理解的是“部件之间如何布置”，而不是“生成一个像某种物体的图像”。因此，如何把结构标签、布局热图或部件图像编码为稳定有效的条件，成为将扩散模型落地到本课题的重要前提。

本课题的扩展方向还与古文字、未登录字符和稀有字符处理紧密相关。近年来，一些研究围绕甲骨文、石刻、古籍手写字符等对象开展识别与数据增强工作，逐步积累了针对特殊字形的建模经验。Gui 等尝试利用扩散模型为手写汉字识别生成训练数据，说明生成方法可以反过来服务识别任务[18]。Xu 等、Zheng 等、Miao 等和 Wu 等分别从持续学习、改进 Swin Transformer、自适应感受野以及自动标注等角度提升古文字识别能力[19-22]。在甲骨文方向，Lin 等将偏旁提取引入识别过程，Wang 等通过无监督判别一致性网络提高类别学习能力，Han 等则从注意力增强和多层特征融合角度改进模型[23-26]。这些工作表明，古文字和未登录字符并不是不可计算的对象，相反，它们具有明显的结构规律和实际应用需求。不过，现有研究重点仍集中在分类与识别，而不是根据部件描述直接生成目标字形。换句话说，相关研究已经证明“这些字值得建模”，但尚未充分回答“如何在部件条件下生成这些字”。

从国内外发展动态来看，相关研究已经出现了较清晰的收敛方向。字符研究越来越强调显式结构表示，单纯把整字视为类别标签的做法正在让位于部件、笔画、层级树和 IDS 等更细粒度表达[4-10]。与此同时，字形生成正从传统 GAN 风格迁移逐步转向更稳定、更细腻的扩散式生成，但其控制信号仍以字体风格为主，尚未形成成熟的“部件 + 结构”条件范式[27-36]。另一边，稀有字符、古文字和未登录字符的研究热度持续上升，这为本课题提供了明确的扩展价值，也提出了更严格的泛化要求：模型不仅要会生成“常见字”，还要能在样本稀缺条件下根据有限构形信息进行外推[18-26]。

综上所述，现有研究已经分别在部件表示、少样本字形生成、扩散模型建模和古文字识别等方面积累了较充分基础，但这些成果尚未自然汇聚为一个完整的“部件与结构条件约束的方块字字形生成”框架[1-36]。识别研究强调可分解表示，却通常不输出字形；字体研究强调风格迁移，却较少直接处理整字组合泛化；古文字研究强调稀有类别处理，却主要停留在识别层面。正因如此，以现代汉字为主任务、以未登录字符和古壮字等对象为扩展方向，构建一套统一的结构化条件字形生成方法，既具有研究空缺，也具备继续深入的现实基础。

## 三、研究内容、研究目标及拟解决的关键问题

### 3.1 研究内容

本课题围绕“部件与结构条件驱动的方块字字形生成”展开，研究内容不再停留在一般性的图像生成，而是聚焦于一种带有明确先验约束的字符生成任务。

课题将把输入条件统一表示为“两个部件 + 一个结构标签 + 可选样式标识”，并在实现层面兼容两种输入模式：文本部件输入适用于现代汉字和标准部件，图像部件输入则服务于未登录字符、古壮字以及不便直接编码的字符样本。围绕这一接口，需要构建目标字图、部件图、结构标签、样式标签之间的一致数据格式，并建立随机切分、整字留出切分和 OOV 扩展小样本集三类实验划分。

考虑到字形任务对结构正确性十分敏感，本课题不会直接从复杂模型起步，而是先建立一个能够稳定训练、可解释性较强的条件 U-Net 基线。模型将从部件图像、部件 ID、结构标签和样式标签中提取条件信息，并在解码过程中输出整字图像与布局相关的辅助预测。借助这一步，可以较早观察模型是否真正学到了左右、上下、包围等空间关系，也为后续扩散模型提供明确的对比对象。

为兼顾生成质量与训练稳定性，课题拟采用“VAE 编码器/解码器 + 潜空间扩散模型”的整体框架。具体来说，先利用 VAE 将字形图像压缩到较低维潜空间，再在潜变量上进行条件去噪生成，使模型在保留结构约束的前提下提升图像细节质量。与基线模型相比，这一阶段的重点不只是出图是否更清晰，更重要的是考察潜空间扩散在结构一致性、组合泛化和条件控制方面是否具有优势。

主任务将在现代汉字上完成，但研究不会止步于已编码字符。课题将进一步选取一小批未登录字符或古壮字样本，采用图像部件输入和 LoRA 微调方式进行轻量适配，验证同一条件接口是否具有向 OOV 场景迁移的能力。同时，课题还将形成训练脚本、评测脚本、命令行推理接口和演示原型，确保方法不仅能写在论文里，也能完成可复现实验与答辩展示。

### 3.2 研究目标

本课题的总体目标，是提出一套适用于方块字的结构化条件生成方法，使模型能够在给定部件和结构约束的条件下输出结构正确、可辨识的字形图像。

数据构造、样本清洗、划分协议、训练加载和推理接口需要形成一致的工程实现，而不是只停留在“输入什么、输出什么”的任务描述层面。只有把这一流程真正闭合起来，后续模型实验才有可比较、可复现的基础。

条件 U-Net 基线需要尽早建立起来。这个目标并不要求基线在所有指标上达到最优，而是要求它能够在小分辨率和标准分辨率下可靠收敛，并给出首批可分析的成功样例和失败样例。只有基线足够稳定，后续主模型的改进才有清晰参照。

整字留出条件下的主模型表现，应当体现出结构约束带来的方法优势。相比基线，主模型需要在结构正确性和组合泛化方面给出更有说服力的结果。这里的“更好”不能只停留在主观观感，而应通过图像相似性指标、结构分类准确率和案例分析共同支撑。

文本部件输入与图像部件输入若能在同一框架下共存，方法才真正具备向 OOV 场景延伸的可能。课题并不把古壮字或未登录字符当作主任务，而是将其作为方法外延来检验模型设计是否具有开放性。若这一点能够成立，该方法就不仅适用于现成汉字字库，也更有机会延伸到字符信息化的边缘场景。

### 3.3 拟解决的关键问题

围绕上述目标，本文拟重点处理几个彼此关联的关键问题。

部件既可以是文本符号，也可以是图像片段；结构既可以是简单标签，也可能需要转化为空间布局先验。若条件表示设计不当，模型容易把不同输入模态割裂开来，导致训练过程不稳定，或者在 OOV 场景下无法迁移。因此，需要在接口层和编码层同时考虑兼容性与可学习性。

字形生成任务中，最容易出现的问题不是图像模糊，而是结构错误，例如左右部件比例失衡、包围关系缺失、部件相互遮挡或整体重心偏移。换句话说，真正困难的地方不在于模型能不能画出轮廓，而在于它是否学会了“怎样组合”。要解决这一问题，模型必须受到明确的布局约束，并在损失函数与辅助监督层面得到针对性引导。

若仅在随机切分上测试，模型很可能通过记忆具体整字获得较高指标，而无法证明它真的具备“部件到整字”的泛化能力。因此，必须把整字留出切分作为重要评测场景，并结合结构探针和定性案例分析，区分“图像看起来像”与“结构确实正确”这两类结果。

未登录字符、古壮字等对象通常缺乏大规模、标准化、可直接训练的数据，这使 OOV 场景下的小样本适配成为框架能否向外延伸的关键检验。若完全重训模型，成本高且不稳定；若仅靠规则拼接，又偏离本课题“由模型生成”的研究目标。因此，需要探索一种参数高效、样本需求较低的适配方式，使扩展实验在方法上自洽、在工程上可行。

## 四、研究方案及可行性分析

### 4.1 总体研究方案

本课题将按照“数据先行、基线先行、主模型推进、扩展验证收口”的思路组织研究过程。整体技术路线并不追求一开始就引入最复杂模型，而是先把研究对象缩小到可控范围：输入限定为两个叶子部件和一个二元结构标签，输出限定为单字灰度图像，主任务先在现代汉字上完成，再把 OOV 场景作为扩展验证。这种做法的好处在于，问题边界足够清晰，数据可自动生成，模型误差也更容易定位。

基于字体渲染和字形拆解表构建结构化样本，是这一路线能够落地的前提。每条样本至少包含目标字、字体样式、目标字图、结构标签、两个部件及其对应图像。为了避免任务规模在早期迅速失控，数据集首版只保留“恰好包含两个叶子部件”的字符样本，不处理多层嵌套字和复杂树结构。结构标签采用统一枚举，包括左右、上下以及若干包围类关系。围绕这些样本，将进一步构造随机切分、整字留出切分和 OOV 扩展样本集，用于分别评估常规拟合能力、组合泛化能力与迁移能力。

前一阶段将构建条件 U-Net 基线，模型通过条件编码器融合部件图像、部件标识、结构标签与样式信息，再结合布局热图生成整字图像。训练损失由重建误差、金字塔感知误差、骨架一致性误差与布局监督共同组成，目的是在图像细节和结构稳定性之间取得平衡。后一阶段转向 VAE 与潜空间扩散模型，先训练 VAE 压缩字形图像，再在潜空间中执行带条件的逐步去噪生成。主模型与基线共用相同的条件接口和结构标签体系，这样可以保证实验对比的公平性，也便于后续把改进来源归因到“生成机制”本身，而不是归因到输入定义变化。

文本部件输入与图像部件输入会保留在同一套条件对象中切换，以便方法在主任务之外仍保留外延能力。主任务默认使用文本部件，因为现代汉字的部件可直接由 Unicode 表示；在未登录字符或古壮字扩展中，则把部件图像直接输入模型，并通过 LoRA 方式进行轻量适配。这样设计的目的，不是追求一次性解决所有复杂字符问题，而是验证方法框架是否具有延展性，是否能从“标准字符”平滑过渡到“非标准字符”。

### 4.2 关键技术路线

数据并不是简单地把整字图像存起来，而是要同时提供部件、结构和样式三类条件信息。为此，需要先对拆解表进行过滤，只保留满足实验设定的样本，再对目标字和部件分别进行统一尺寸、统一灰度、统一居中的渲染与保存。对于 OOV 扩展部分，还需要提供图像部件输入模板，以便外部样本进入同一数据协议。

基线模型中，部件图像、部件 ID、样式标签和结构标签将进入共享条件编码器，再通过空间广播与布局热图共同驱动 U-Net 解码。这样做的考虑是，文字级别的部件 ID 有利于模型建立稳定的符号语义，图像级别的部件输入则有利于处理未登录字符；二者结合可以减少模型完全依赖某一模态带来的脆弱性。

直接在像素空间进行扩散，虽然理论上可行，但在当前任务下会带来更高的训练成本和更长的迭代周期。采用 VAE 先压缩，再做潜空间扩散，一方面能够降低计算开销，另一方面也有利于在较小实验资源下进行超参数搜索。与此同时，布局热图不只是辅助可视化，而是作为显式结构先验输入到去噪网络中，用来约束模型在每一步都关注部件的空间摆放，而不是只追求局部纹理。

仅使用 SSIM 或 LPIPS 这样的视觉指标并不足以覆盖字形任务的关键风险，因此课题将引入结构探针或结构分类准确率，用来衡量生成结果在结构层面是否与条件一致。除此之外，还将固定一批代表性案例，包括直观样本与容易混淆的难例，对生成结果进行定性分析，特别关注“视觉清晰但结构错误”“结构正确但风格不稳”“部件齐全但重心失衡”等典型失败模式。

### 4.3 可行性分析

本课题的主数据来源并不依赖难以获取的人工标注，而是可以通过公开字体和拆字表自动构造。相比需要大规模人工描字或复杂矢量标注的研究方向，这种数据构建方式更适合硕士阶段的时间与资源条件。由于 V1 版本只处理两个部件和一个结构标签，数据清洗规则也较为明确，样本协议具备较强的可操作性。

研究路线采用“先基线、后扩散”的安排，也有助于降低整体风险。即便扩散模型训练周期较长，基线模型仍可先给出一套完整实验闭环，保证课题不会因为主模型训练不稳定而整体停摆。与此同时，主模型的条件接口与基线保持一致，也使得实验对比和模块替换更直接，不需要在后期推翻前期数据与代码设计。

现有原型已经具备较明确的代码骨架，包括数据构造脚本、基线训练脚本、VAE 训练脚本、扩散训练脚本、结构评测脚本、推理接口和演示入口。这意味着课题并不是从零开始摸索，而是在已有研究原型的基础上继续深化方法与实验。这样的基础有助于把更多精力投入到模型设计、实验分析和论文论证，而不是反复消耗在底层工程搭建上。

当然，课题仍面临几个需要提前应对的风险。扩散模型在小规模数据上可能出现过拟合或收敛偏慢，结构监督不足时模型也可能偏向记忆整字纹理，OOV 扩展样本过少则会导致迁移结果波动。相应的应对策略已经比较明确：先以较低分辨率跑通训练，再逐步提升到标准分辨率；在损失函数与布局监督中显式强调结构约束；在扩展实验中采用 LoRA 等参数高效方法，避免大规模重训带来的不确定性。总体来看，问题定义、数据来源、模型路线与现有工程条件之间具有较好的匹配关系，课题具备较强的实施可行性。

## 五、论文选题的特色与创新之处

现有相关研究往往把字形问题分散到识别、字体迁移、风格合成或古文字分类等不同任务中，而本课题试图把“部件可分解性”和“图像可生成性”放回同一个研究对象中讨论。这样收口之后，面对的就不再是边界松散的文生图问题，也不是依赖规则拼接的补字工具，而是一个输入条件明确、结构约束清楚的生成任务。问题一旦被限定在这个层面，研究重点也就能从泛化的视觉生成转向更具体的结构建模与组合泛化分析。

这一框架能否成立，很大程度上取决于条件接口是否统一。`文本部件输入` 适合现代汉字的数据构造与词表管理，`图像部件输入` 则更贴近未登录字符、古壮字和异体字的实际使用场景。把两者放进同一个条件对象后，模型面对的不是两套彼此割裂的方法，而是一套可以在标准字符与非标准字符之间切换的生成机制。也正因为如此，扩展验证不需要被拔高为新的主任务，却足以说明该方法是否具有向外迁移的能力。进一步采用 `LoRA` 做小样本适配，也让这种验证停留在方法可行性层面，而不会把论文重心拖离现代汉字主线。

在模型层面，本课题并不满足于给出一个全局条件后让网络自行摸索布局，而是把空间关系本身作为必须显式进入训练的约束信息。为此，结构标签之外还会引入 `布局热图` 和辅助 `结构监督`，把“左右”“上下”“包围”等关系转化为模型能够直接利用的空间先验。这样的处理不是为了让结果看起来更复杂，而是因为字形任务对结构错误极其敏感。模型是否知道部件该放在哪里、比例如何分配、轮廓怎样衔接，往往比局部纹理是否足够细腻更先决定生成结果能不能被读出来。

评测环节同样围绕这个核心问题来设计。若只在随机切分上报告图像指标，模型完全可能依靠记忆整字分布获得不错结果，却无法说明它是否真正掌握了部件组合规律。因此，`整字留出切分` 会被放在更关键的位置，并与结构探针、结构正确性分析和失败类型归纳配合使用。这样得到的结论更接近本课题真正关心的问题：模型是在复现训练集中见过的字形，还是在未见组合上仍能依据部件与结构进行生成。

因此，本文的创新并不在于宣称提出一种能够全面替代现有字库或彻底解决所有稀有字符问题的通用系统，而在于围绕“部件与结构条件如何驱动字形生成”这一核心问题，搭建了一套更可解释、可扩展、可评测的研究框架。构形知识、条件生成和组合泛化原本分散在不同方向中，在这里被收拢到同一任务之下；统一接口、结构监督和扩展验证则把这条方法链条进一步闭合起来。这样的创新路径相对稳妥，也更符合硕士论文对问题聚焦、论证完整和方法可落地性的要求。

## 六、研究计划及预期研究结果

本课题拟按一年左右的研究周期推进，时间安排遵循“先收口问题，再完成实验，最后集中打磨论文”的原则。结合当前原型基础与硕士论文常规进度，可将研究计划概括为以下几个阶段。

1. `2026年4月—2026年6月`：完成文献梳理、任务定义收口和数据协议设计，明确部件集合、结构标签、样本筛选规则及实验划分方式；同步补齐数据构造与样本检查脚本，形成可重复的数据准备流程。
2. `2026年7月—2026年9月`：完成基线模型训练与初步分析，先在较低分辨率下跑通训练、保存、加载、推理与评测全链路，再切换到标准分辨率获取第一版可展示结果；整理成功样例与失败样例，为主模型设计提供依据。
3. `2026年10月—2026年12月`：完成 VAE 与潜空间扩散主模型训练，围绕条件融合位置、布局监督方式、损失权重等因素开展消融实验；在随机切分与整字留出切分上完成核心对比，形成论文主体实验材料。
4. `2027年1月—2027年2月`：开展 OOV 扩展验证与系统演示整合，选取少量未登录字符或古壮字样本进行图像条件适配实验，完善命令行推理、演示界面和案例可视化。
5. `2027年3月—2027年5月`：完成论文写作、图表整理、结果复核和答辩准备，重点补充方法图、实验表、失败分析和演示脚本，确保论文文本与实验材料保持一致。

在预期研究结果方面，本文希望形成一套面向方块字结构化条件生成的研究原型，覆盖数据准备、模型训练、模型评测、命令行推理和演示程序等环节；完成条件 U-Net 基线与潜空间扩散主模型的系统对比，明确不同模型在图像质量、结构正确性和组合泛化上的表现差异；建立一套适合本任务的评测协议，尤其是在整字留出场景下对结构正确性进行分析；通过小规模 OOV 扩展实验验证统一条件接口的可迁移性；最终完成硕士学位论文撰写，并在条件允许的情况下整理出可进一步投稿或继续扩展的研究材料。

若研究按计划推进，预期论文最终能够给出一个较为清楚的结论：在方块字场景中，把部件信息与结构先验显式引入生成模型，不仅有助于提升字形图像质量，更重要的是能够改善模型对未见组合的生成能力。这一结论若能通过实验得到支持，将为后续字符生成、稀有字信息化处理以及相关数字人文应用提供具有延展性的技术基础。
